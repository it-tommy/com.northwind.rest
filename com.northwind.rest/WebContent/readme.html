Part 1

Read Me

Description: Part 1 consist of the basic CRUD operations that are used in a typical database driven web application. This RESTful API is of the familiar fictitious Northwinds Trading Company. The following sections of Part 1 will describe the CRUD operations and how they are used in conjunction with each RESTful web service call. The returning data transport mechanism is JSON unless other wise specified. I will try to incorporate some xml in the future. I have built a generic UI using JQuery that that should help visaulize the functionality of the API. The Restful base url is com.northwind.rest Also, I have prefixed the url's with a descriptive version identifier. Example...{base RESTful base url} /v1/{general data description}. Part 2 will show more of a implemented business logic vs. basic CRUD operations.
~Tommy Smith

Section #1

Description: This section shows the basic "Read" operation of CRUD for this API. The HTTP method used for this operation is the GET method.

Example 1.1 Read all the products from the products table. http://69.243.204.126:91/com.northwind.rest/api/v1/products/inventory/

Example 1.2 Read one of the products from the products table. This request uses a Query Parameter which further qualifies the querying record. In this case the product Id number is 31. http://69.243.204.126:91/com.northwind.rest/api/v1/products?product_id=31

Note: If your request URL is in this form http://69.243.204.126:91/com.northwind.rest/api/v1/products You will receive a status code of 400 which signifies a Bad Request. The logic to handle product request in which the product id does not exist has been added too. http://69.243.204.126:91/com.northwind.rest/api/v1/products?product_id=1151There is no product were product_id=1151

Example 1.3 Read one of the products from the products table. This request uses a Path Parameter which further qualifies the querying record. In this case the product Id number is 27. http://69.243.204.126:91/com.northwind.rest/api/v1/products/27

Note: Again, if your request URL is in this form http://69.243.204.126:91/com.northwind.rest/api/v1/products You will receive a status code of 400 which signifies a Bad Request. The logic to handle product request in which the product id does not exist has been added too. http://69.243.204.126:91/com.northwind.rest/api/v1/products/201There is no product were product_id=201
